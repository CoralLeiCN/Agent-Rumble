# Agent Rumble Two-Minute Demo

This script demonstrates the implemented local product. It does not present the
planned hosted card-generation service or marketplace publication as complete.

## 0:00–0:20 — The Problem

> Agent projects are difficult to compare. The useful details are scattered
> across source code, documentation, configuration, and examples, and project
> descriptions rarely use the same structure.

> Agent Rumble turns that evidence into standardized Agent Project Cards so
> people and agents can discover projects and understand their trade-offs.

## 0:20–0:45 — Search the Catalog

Open Agent Rumble and search for a concrete need. Point out that the results come
from the complete backend catalog of preprocessed cards rather than a live model
call or a small frontend sample.

Show the interpreted request, project roles, match explanations, constraints,
analyzed revisions, and review dates.

## 0:45–1:15 — Compare and Verify

Shortlist two or three projects and open the comparison. Show that Agent Rumble
keeps shared fields available, highlights material differences, and preserves
unknown or unanalyzed states instead of treating them as failures.

Open one consequential value and show its Claim, verification status,
confidence, pinned source revision, locator, and evidence excerpt.

## 1:15–1:40 — Enter Rumble Arena

Select OpenAI Agents SDK and LangGraph for the prepared evidence-backed matchup.
Show the contextual rounds, then start the playable arcade mode. Point out that
the fighters have equal gameplay budgets: project evidence themes their moves,
but it does not determine damage or declare a better project.

## 1:40–2:00 — Explain Codex and Close

> Codex is the analysis harness behind the Agent Project Card skill. It helps us
> inspect repositories, structure material findings as claims and evidence, and
> validate versioned `project-card.yaml` artifacts before they enter the catalog.

> Agent Rumble makes that prepared intelligence searchable, comparable, and
> reusable, so every decision does not have to begin with another repository
> investigation.
