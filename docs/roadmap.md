# Product Roadmap

**Status:** Proposed

The capabilities below are possible post-MVP directions. They are not active requirements and do not expand the [MVP scope](specification/06-mvp-scope-and-evaluation.md#19-mvp-scope).

Explicitly requested capabilities and implementation work that have been
deliberately postponed are tracked separately in the
[deferred backlog](backlog.md).

## Multi-Project Comparison

Generate structured comparison tables with:

* Shared capabilities
* Unique capabilities
* Architectural differences
* Maturity differences
* Integration trade-offs
* Best-fit scenarios

## Agent Architect

Translate a business use case into required architecture layers and recommend relevant projects.

## Project Relationship Graph

Model:

* Dependencies
* Integrations
* Alternatives
* Complementary projects
* Forks
* Ecosystem relationships

## Change Intelligence

Detect:

* New capabilities
* Breaking changes
* Removed features
* Dependency changes
* License changes
* Maintenance decline
* Security-relevant changes

## Dynamic Verification

Run setup steps, examples, tests, and benchmarks inside isolated environments.

## Organization-Specific Cards

Add internal fields such as:

* Approved status
* Internal owner
* Security review
* Existing deployments
* Internal expertise
* Procurement status
* Preferred alternatives

## Landscape and GTM Analysis

Analyze:

* Category position
* Differentiation
* Competitive strengths
* Missing capabilities
* Adoption barriers
* Ecosystem opportunities
* Potential partnerships

## Ecosystem Trend Intelligence

Aggregate comparable Agent Project Cards to analyze:

* Model-provider support
* Agent SDK and framework usage
* Technology and infrastructure adoption
* Capability coverage and change over time

Trend outputs should preserve their cohort, time range, denominator, metric definition, data completeness, and links to the supporting cards. They should not generalize from missing or unanalyzed card fields.
